Trouw DS v17 · Form Patterns

Showcase complementar ao trouw-ds-showcase-v16.html. Documenta os componentes que emergiram do cadastro-cliente v7 e foram reconciliados na proposta v17 (tokens-reconciliation-v17.md). Foco: telas de admin, cadastro, wizard, dashboards em densidade form.

Fonte: Plus Jakarta Sans Densidade: form Tokens DTCG W3C 1.0 WCAG 2.2 AA

Visão geral

read first

A v17 não substitui a v16 — complementa. A v16 cobre a Torre Operacional (densidade ops, tipografia a partir de 9px, cards densos). A v17 cobre telas de cadastro, admin e wizard (densidade form, tipografia a partir de 11px, componentes de formulário).

O que muda em nomes de tokens

  • --neutral-* → --gray-* (13 shades, inclui --gray-150 novo)
  • --space-* → --s-* (escala numérica direta em px)
  • --text-* → --fs-* (font-size, evita colidir com --text-default)
  • --radius-* → --r-* · --elev-* → --sh-*
  • --s1/s2/s3 → --bg-surface/subtle/sunken
  • --t1/t2/t3 → --text-default/muted/subtle (t4 removido — falhava AA)
  • --blue/red/amber/green → --primary/feedback-danger/warn/ok
  • Escala --sev-info/low/med/high/crit promovida a oficial
Legacy aliases: durante a migração, os nomes v16 seguem funcionando via ds-v17-legacy-aliases.css (gerado no passo B do plano). Remover aliases em v18.

Accordion

new

Blocos colapsáveis com header + ícone + meta + status. Usado em cadastros multi-seção (ex: Step 1 do cadastro de cliente). Toggle via data-state="open".

Radio Card

new

Radio button expandido com título + descrição. Use quando a diferença entre opções precisa de explicação (não apenas um label). data-selected="true" aplica border + background do primary.

Chip Multi

new

Seleção múltipla inline com estética de pill. Alternativa a checkboxes quando os itens são curtos e cabem numa linha.

Se nenhuma for marcada, o usuário vê todas as filiais.

Role Badge

new

Identifica a função do usuário no sistema. 4 variantes via data-r. Palette mapeia severidade do papel: Admin (danger) > Gerente (primary) > Operador (info) > Visualizador (muted).

Status Dot

new

Indicador de status com dot colorido + label. 3 variantes: ativo (ok), pendente (warn), inativo (muted).

KPI Strip · Ops

new

Strip de métricas filtráveis do dashboard operacional. Migração do padrão da Torre de Controle v2_85 (DS v16) para tokens v17 (--feedback-*, --bg-*, --text-*). Três grupos semânticos — Operação, Prazo, Atenção — com valor em tabular-nums, accent bar esquerda colorida por tone, e delta badge opcional (↑ danger-soft / ↓ ok-soft). Cada card é clicável como filtro dimensional.
